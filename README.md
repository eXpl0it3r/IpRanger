# IpRanger

TCP traffic monitoring and blocking tool for Debian/Ubuntu servers.
Monitor incoming connections, enrich IP data via RDAP, pull threat intelligence from block lists, and manage iptables/ipset rules - all from a modern web UI.

## Features

- **Live monitoring** - polls `ss -tnp` every 10 s, deduplicates TCP sessions, and tracks cumulative + live connection counts per IP
- **RDAP enrichment** - auto-resolves org, network, ASN, and country for each IP; skips RFC-private ranges
- **Block lists** - fetches FireHOL, Spamhaus DROP, Emerging Threats, and Tor exit nodes on a 24 h schedule; ranges are synced into ipset
- **ipset / iptables** - create/flush/sync sets and install DROP rules without hand-editing; supports blocking whole RDAP networks
- **Friendly IPs** - whitelist IPs or CIDR ranges; all RFC-private ranges are pre-seeded
- **Country blocking** - optional per-country blocking based on RDAP country codes
- **Basic auth** - configurable username/password to protect the web UI
- **Web UI** - dashboard, statistics, network explorer, log viewer, blocked IPs, bad-IP explorer, and settings via Flask + HTMX + Tailwind CSS

## Requirements

- Debian / Ubuntu server
- Python 3.10+
- `ipset` and `iptables` installed
- Root privileges (needed for ipset/iptables)

## Installation

```bash
git clone https://github.com/you/ipranger.git
cd ipranger
pip install -r requirements.txt
```

Edit `config.yaml` before starting - at minimum set:

```yaml
server:
  secret_key: "change-me"   # used for session signing
  auth:
    username: "admin"
    password: "your-password"  # leave empty to disable auth
```

## Usage

```bash
sudo python run.py
```

Open `http://<server-ip>:5000` in your browser.

## Configuration

All settings live in `config.yaml`:

| Key                                | Default                     | Description                                        |
|------------------------------------|-----------------------------|----------------------------------------------------|
| `server.host`                      | `0.0.0.0`                   | Bind address                                       |
| `server.port`                      | `5000`                      | HTTP port                                          |
| `server.debug`                     | `false`                     | Flask debug mode                                   |
| `server.secret_key`                | `"change-me-in-production"` | Session signing key - **change this**              |
| `server.auth.enabled`              | `true`                      | Enable HTTP basic auth                             |
| `server.auth.username`             | `admin`                     | Basic auth username                                |
| `server.auth.password`             | `"change-me"`               | Basic auth password - **change this**              |
| `database.path`                    | `ipranger.db`               | SQLite database file path                          |
| `monitoring.interval_seconds`      | `10`                        | How often to poll connections                      |
| `monitoring.flag_threshold`        | `500`                       | Connection count that flags an IP                  |
| `blocklists.update_interval_hours` | `24`                        | Block list refresh interval                        |
| `rdap.cache_ttl_hours`             | `168`                       | How long RDAP results are cached (7 days)          |
| `rdap.lookup_delay_seconds`        | `1`                         | Delay between RDAP requests to avoid rate-limiting |
| `ipset.set_name`                   | `ipranger_blocked`          | Name of the ipset set                              |
| `ipset.auto_block`                 | `false`                     | Automatically block flagged IPs                    |
| `ipset.auto_block_threshold`       | `1000`                      | Connection count that triggers auto-block          |
| `ipset.persist`                    | `true`                      | Persist ipset rules across reboots                 |
| `countries.blocking_enabled`       | `false`                     | Enable country-level blocking                      |
| `countries.blocked_countries`      | `[]`                        | ISO 3166-1 alpha-2 codes to block                  |

### Block list sources

Six sources are pre-configured. Enable or disable them in `config.yaml` under `blocklists.sources`, or manage them live from the **Settings** page.

| Name               | Type | Enabled by default |
|--------------------|------|--------------------|
| `firehol_level1`   | CIDR | ❌                  |
| `firehol_level2`   | CIDR | ❌                  |
| `emerging_threats` | IP   | ✅                  |
| `spamhaus_drop`    | CIDR | ✅                  |
| `cinsscore`        | IP   | ✅                  |
| `tor_exit_nodes`   | IP   | ❌                  |

## Web UI pages

| Page           | Description                                                                                                                                     |
|----------------|-------------------------------------------------------------------------------------------------------------------------------------------------|
| **Dashboard**  | Overview cards (total IPs, cumulative connections, **live open connections**, blocked, flagged, block list, friendly) - auto-refresh every 10 s |
| **Statistics** | Searchable, sortable per-IP table with RDAP data and block/friendly actions                                                                     |
| **Networks**   | RDAP-resolved networks/prefixes grouped by CIDR, sortable by IP count; expandable drilldown and per-network block action                        |
| **Blocked**    | Manually and automatically blocked entries; add/remove from here                                                                                |
| **Bad IPs**    | Browser for block list entries, filterable by source                                                                                            |
| **Settings**   | Block list management, friendly IPs, ipset status/sync, and **Clear IP History** (resets counts while retaining RDAP data)                      |
| **Logs**       | Live ring-buffer log viewer with level/search filters                                                                                           |
